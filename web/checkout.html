What you get

Pay → Pass flow
1. Create invoice

Client requests a BOLT11 invoice from the API.

2. Pay with any wallet

LDK node confirms settlement and emits PaymentClaimed.

3. Issue access token

Service signs a payload with Ed25519 and returns a QR.

4. Verify offline

Gate verifier checks signature, expiry, and amount locally.

FAQ
What wallets are supported?

Any Lightning-enabled wallet that can pay a BOLT11 invoice will work.

Do you store attendee data?

No. Access is derived from payment settlement and signed tokens only.

Can I use my own Lightning node?

Yes. The demo runs on LDK; production can run on your infrastructure.

How fast is the pass delivered after payment?

Within seconds of settlement. The access token replaces the invoice immediately.

Signing up

HASKE sign-ups are open to any organizer building privacy-first access flows. Each workspace provisions a Lightning node profile and a dedicated pass namespace.

Eligibility

To activate a HASKE workspace, you’ll need:

  • A Lightning wallet capable of paying BOLT11 invoices.
  • A verified contact method for event support.
  • An event or product use case ready for onboarding.
Admissions

We admit teams on a rolling basis. If demand exceeds capacity, we prioritize applications with upcoming events and clear deployment timelines.

Contributions are public

HASKE access passes are anonymous but auditable. Each pass can be verified without revealing attendee identity, keeping check-ins fast and private.

Rules

Organizers must honor refund requests, avoid reselling passes, and ensure policies are communicated before payment.
